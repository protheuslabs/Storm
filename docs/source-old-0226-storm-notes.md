## Core Emphasis

This note restates Storm's original emotional center: latent creative ideas are one of humanity's largest untapped resources, and Storm should make creative ideas much easier to manifest.

The note frames Storm as a system where creative input can compound like an investment. If a person's idea, prompt, refinement, or other input helps create value for others, some of that value should flow back to them.

## Creative Seed

The note uses the idea of a seed:

- Someone plants an idea.
- The crowd helps build it.
- Contributions are tracked.
- Value created downstream flows back to contributors.
- The seed can compound over time.

This reinforces that Storm is not only about paid tasks. It is also about preserving and growing ideas that would otherwise disappear.

## Residual Value

The note strongly emphasizes permanent or long-lived residual value.

Example:

- A person has an idea for a book.
- They plant the seed.
- The crowd helps build the book.
- Contributors receive residual value based on how much they added.

This supports the long-term residual-claim model, but it should remain outside MVP until direct task payout, attribution, ownership, and dispute systems are proven.

## Phone-Sized Work

The note describes tasks divided small enough that someone could complete one on a phone in about five minutes.

This strengthens the microtask direction, but it also increases review and coordination overhead. MVP tasks should probably be small and clear, but not necessarily five-minute microtasks until the review system can handle that granularity.

## Economic Frame

The note repeats the value split goal:

- Contributors keep roughly 95% of the value they generate.
- The platform keeps roughly 5%.

It also frames Storm as a challenge to the current economic model where companies capture most of the value generated by workers.

## Practical Takeaway

This source sharpens Storm's product thesis:

Storm should be an idea-to-value engine where creative seeds are captured, decomposed, built by crowds, attributed fairly, and paid back over time.

Near-term product still starts with:

Project -> Task -> Claim -> Submit -> Review -> Ledger -> Reputation.

But long-term product should preserve:

- Idea seeds.
- Creative input attribution.
- Residual claims.
- Microtask decomposition.
- Low-extraction platform economics.
- A path for abandoned ideas to become real outputs.
